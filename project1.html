The Basics of the Web and HTML

The World Wide Web is a collection of HTML documents. HTML stands for Hypertext Markup Language.
Concepts of the Web:
  • Browser - a program that runs on your computer to display files found on the web
  • Internet - the world's largest computer network
  • HTTP - the main protocol of the web
  • Servers - computers that host the files that make up the web
HTML is made up of:
  • Text content - what you see
  • Markup - what it looks like
  • References to other documents - i.e. images, videos
  • Links to other pages
Tags and Elements

HTML documents are made of HTML elements. An HTML element refers to everything within a set of opening and closing tags. Some elements also have attributes.

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Web Pages, Documents, and Structure

  • HTML (Hyper Text Markup Language) files controls the structure of a web page.
  • CSS (Cascading Style Sheets ) files controls the style of a page (how it looks)
  • Javascript is the interactive component.
HTML is the language of the web. It has specific syntax and rules. The basic word in the language of HTML is a tag. The browser turns HTML tags into elements that form a tree (elements can have other elements within). It knows how to do that because of the DOM (Document Object Model) which is the standard convention for representing and interacting with HTML elements. Every element in a web page is a box.

CSS use specific syntax and rules that change how elements look in the page, i.e. font-size, color, border, etc.

There are three ways to include CSS Styling in a web page.
  1. Write CSS in the "head" section. This method is good for very small projects.
  2. Link the HTML page to a separate CSS file. This adds another step but is more organized when working on larger projects.
  3. Write the style "inline" with the HTML page. This should be avoided because it leads to lots of repeated codes. The same styles are copied and pasted into many elements. When changes are required, rewriting the codes can lead to errors. Repetition should be avoided.
Creating & Designing a Page:
Write the code, compare against design mock-up, refine code until it looks like the design. Get the structure of the page right. Visualize the boxes. work on the tags, size and position of the boxes. Use developer tools to refine the page without actually changing the codes until it meets the specification.
